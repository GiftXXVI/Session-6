Organizing API Endpoints

Principle 1

  • Organize URLs using the resource being accessed or modified.
  • The pathname should contain the name of the entity that is being retrieved or manipulated.
  • Examples: 
    								
									/students
									/students/1
									/projects
									/modules

Principle 2

  • Maintain a consistent naming scheme.
  • Follow the following guidelines:
    • Use plural nouns.
    • Limit the number of segments to reduce the length of the addresses.

Principle 3

  • Do not include the action in the URL.
  • Instead, let the HTTP Message describe the action.
  • Recall that HTTP Methods map to CRUD operations. For example:
    • GET retrieves resources.
    • POST submits new data to the server.
    • PUT updates existing data.
    • DELETE removes data.

Summary

  • Here are a few examples of endpoints and methods:
  • To get a list of students: 
    								
									GET /students
  • To create a new student: 
    								
									POST /students
  • To update a student: 
    								
									PATCH /students/1
  • To delete a student: 
    								
									DELETE /students/1
