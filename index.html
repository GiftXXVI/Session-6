Organizing API Endpoints üßÆ

Include the resource name in the URL

  • Organize URLs using the name of the resource being accessed or modified.
  • Specifically, use a noun representing the name of the resource, do not use a verb describing the action.
  • Examples:
    • Good: 
      /students
    • Bad: 
      /get_students

Keep a consistent naming scheme

  • Use plural nouns for the resource name within all URLs.
  • Use an id for URLs that are used to retrieve or modify a single object.
  • Examples:
    • Good: 
      /students/1
    • Bad: 
      /student/1

Limit the number of URL segments

  • As a guideline, do not exceed 3 segments in the URL.
  • Examples:
    • Good: 
      /students/1/projects
    • Bad: 
      /students/1/projects/2/tasks

Use the HTTP Method to define the action

  • Recall that HTTP Methods map to CRUD operations.
  • Examples:
    • Retrieve Students: 
      GET /students
    • Create Student: 
      POST /students

Use the HTTP Method to define the action (continued)

  • Recall that HTTP Methods map to CRUD operations.
  • Examples:
    • Modify Student: 
      PATCH /students/1
    • Delete Student: 
      DELETE /students

Examples of Flask Endpoint Definitions.

  • Here are a few routes to demonstrate these principles:
  • Retrieve Students: 
    @app.route('/students',methods=['GET'])
  • Create Student: 
    @app.route('/students',methods=['POST'])

Examples of Flask Endpoint Definitions.(continued)

  • Here are a few routes to demonstrate these principles:
  • Modify Student: 
    @app.route('/students/',methods=['PATCH'])
  • Delete Student: 
    @app.route('/students/',methods=['DELETE'])

CORS - Cross-Origin Resources Sharing

  • CORS is the process of sharing resource(s) from between different origins(addresses).
  • Same-origin policy
    • Web Applications are not allowed by default to share resources with another application on a different origin (address) for security reasons.

Cross-Origin

  • Here are a few examples of Cross-Origin scenarios:
    • Different domains: 
      https://udacity.com
      and 
      https://github.com

Cross-Origin

  • Here are a few examples of Cross-Origin scenarios:
    • Different subdomains:

Cross-Origin

  • Here are a few examples of Cross-Origin scenarios:
    • Different ports:
